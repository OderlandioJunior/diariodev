Propriedades para elementos

No assunto sobre elementos flexíveis, temos o order, align-self e o trio flex (flex-basis, flex-shrink e flex-grow).

Order

Assimilando um ID para um elemento HTML, conseguimos mudar sua posição de exibição no eixo principal (main-axis) nas CSS, diferente da pseudo-classe ::after que cria um conteúdo "virtual" em um conteúdo já existente.

Forma de uso

Essa propriedade pode receber valores numéricos e possui como valor padrão o número 0, onde são mostrados respectivamente na linha de exibição por exemplo flex-flow: row wrap de acordo com os Numeros Inteiros, sendo, os números negativos mostrando os primeiros itens, seguindo logo após o 0 e assim finalizando a lista de exibição com os números positivos. Uma OBSERVAÇÃO é que, dado o valor 0 a dois elementos, a exibição será de acordo às suas respectivas posições no documento HTML.

Align-Self

O align-self funciona alinhando o itens no eixo cruzado (cross-axis), porém, diferente do align-items e align-content, essa propriedade atua de forma individual para os itens.

Forma de uso

Os valores para essa propriedade são:

auto
Herda a caracteristica do conteiner
flex-start
Alinha o item perto do cross-start
flex-end
Alinha o item perto do cross-end
center
Calcula automaticamente o centro do conteiner e alinha o item
stretch
Estica o item para ocupar 100% da altura do conteiner

Flex-basis, flex-shrink e flex-grow

Partindo para o trio flex, temos o flex-basis dita o tamanho máximo que o elemento terá alinhado com o main-axis, ou seja, esse tamanho poderá ser a altura ou a largura. O flex-shrink dita se o elemento poderá encolher e o flex-grow se pode crescer dentro do container

Formas de usos

O trio flex aceita valores númericos e pode ser simplificado com um shorthand flex nas CSS. Esse shorthand por sua vez é respondido respectivamente por flex-grow e o flex-shrink com valores com 0 para não e 1 para sim mas não são exatamente propriedades binárias e finalizando com flex-basis com unidades de medidas.

Sobre o flex-basis, seu valor padrão é auto, esse valor faz com que os itens aumente o tamanho para comportar seu contéudo individual. Porém, adicionando um valor numérico à essa propriedade, os itens terão o respectivo tamanho até que todos caibam dentro do conteiner, e, a partir do momento em que não haja mais espaço para todos, eles irão acompanhar o flex-flow, reduzindo seu tamanho e quebrando o conteúdo ou levar o último item para o cross-axis

O flex-shrink possui como padrão o valor 1 e o flex-grow como 0.
Uma OBSERVAÇÃO para os valores de flex-shrink e flex-grow é que, dado o valor 1 para um elemento e o valor 2 para outro, os elementos irão calcular automaticamente o espaço do conteiner e vão usar o valor maior como multiplicador para encontrar uma distribuição. Por exemplo: item A com flex-grow de 1 e item B com flex-grow de 3 em um conteiner, o item B terá 3x mais tamanho ao crescer a janela do que o item A. Da mesma forma, funciona com flex-shrink.

Voltando a falar do shorthand, é possível simplificar ainda mais com valores "únicos", por exemplo: flex: 0 1 auto pode ser escrito como flex: initial pois são todos os valores padrões das propriedades em formas separadas // flex: 0 0 auto significa que o item é totalmente não flexível, corespondido como flex:none // flex: 1 1 auto dá ao item livre movimento, podendo crescer e diminuir e ter seu limite natural por conta do conteúdo, igualando seu resultado a um flex: auto // Por fim, temos o shorthand com apenas um valor, por exemplo: flex: 3, isso significa que será aplicada a mudança somente no flex-grow, seguindo com a ordem padrão das outras propriedades.
